ShanPaul Stationary System 🖊️💻
A comprehensive Full-Stack Management System designed for stationery businesses. This project integrates a RESTful API, a Windows Desktop Application, and a Responsive Web Interface to handle inventory and sales seamlessly.

🚀 Features
Full CRUD Operations: Manage stationery inventory (Create, Read, Update, Delete) through a centralized API.

Real-time Stock Management: Logic-driven system that automatically updates stock levels upon new orders or cancellations.

Cross-Platform Accessibility:

Desktop App: Built for internal store operations using Windows Forms.

Web Portal: Responsive interface for customers to browse products and place orders.

Relational Database: Structured MySQL database ensuring data integrity between 'Barang' (Items) and 'Pemesanan' (Orders).

🛠️ Tech Stack
Backend: ASP.NET Core Web API

Desktop: C# .NET Windows Forms

Frontend: HTML5, CSS3, JavaScript (ES6+)

Database: MySQL with Entity Framework Core

Tools: Visual Studio, Git, Postman (API Testing)

🏗️ Architecture
The system follows a Service-Pattern architecture to separate business logic from the controllers, making the code scalable and maintainable:

Controllers: Handle HTTP requests and responses.

Services: Contain the core business logic and database interactions.

DTOs (Data Transfer Objects): Ensure secure and efficient data transfer between layers.
